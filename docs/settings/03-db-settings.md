# 03. 数据库设置

## 数据库路径

### 默认路径

- **Windows**：`%APPDATA%\KkbarDB`
- **macOS**：`~/Library/Application Support/KkbarDB`

### 自定义路径

如需更改数据库存储位置：

1. 打开设置面板
2. 找到 **数据库路径** 选项
3. 点击 **设置** 按钮选择新目录
4. 选择 **创建新数据库**（空白数据库）或 **复制现有数据**

### 建议

首次使用时建议设置自定义目录，便于：
- 备份和管理数据文件
- 多设备间同步数据

---

## 数据导出

### 方式一：导出单个分类

在设置面板中：

1. 选择要导出的分类
2. 点击分类右侧的导出按钮
3. 选择保存位置和文件名

导出文件包含该分类及其所有按钮数据（`.kbar` 格式）。

### 方式二：导出 UI 设置

在设置面板中：

1. 找到 **导出配置** 按钮
2. 选择 **UI 设置** 选项
3. 选择保存位置

导出文件包含界面主题、图标大小、布局等设置（`.json` 格式）。

### 方式三：导出完整数据库

在设置面板中：

1. 找到 **导出配置** 按钮
2. 选择 **完整数据库** 选项
3. 选择保存位置

导出文件包含所有分类、按钮、图标、UI 设置等完整数据。

---

## 数据导入

### 导入分类数据

1. 在设置面板中点击 **导入配置**
2. 选择要导入的 `.kbar` 文件
3. 数据会合并到现有分类中

### 导入 UI 设置

1. 在设置面板中点击 **导入配置**
2. 选择要导入的 UI 设置文件（`.json`）
3. 确认后 UI 设置会被覆盖

### 导入完整数据库

1. 在设置面板中点击 **导入配置**
2. 选择要导入的完整数据库文件
3. 确认后所有数据会被覆盖

---

## 数据备份建议

- 定期导出完整数据库作为备份
- 重要配置单独导出 UI 设置
- 可导出单个分类与他人分享

---

## 核心特点

- **本地存储**：数据存储在本地文件系统
- **自定义路径**：可更改数据库存储位置
- **多种导出**：支持导出单个分类、UI 设置、完整数据库
- **格式标准**：.kbar 格式用于分类，.json 格式用于设置
- **数据合并**：导入分类数据时与现有数据合并

## 适用场景

- 备份数据库防止数据丢失
- 迁移到新电脑时转移配置
- 与他人分享单个分类的按钮配置
- 仅迁移 UI 设置保留按钮配置
