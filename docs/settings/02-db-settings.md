# 💾 数据库设置

## 数据库路径

### 默认路径

- **Windows**：`%APPDATA%\KkbarDB`
- **macOS**：`~/Library/Application Support/KkbarDB`

### 自定义路径

如需更改数据库存储位置：

1. 打开设置面板
2. 找到 **数据库路径** 选项
3. 点击 **设置** 按钮选择新目录
4. 选择 **创建新数据库**（空白数据库）或 **复制现有数据**

建议首次使用时设置自定义目录，便于备份和管理数据文件、多设备间同步数据。

---

## 数据导出

:::details 导出单个分类
在设置面板中：

1. 选择要导出的分类
2. 点击分类右侧的导出按钮
3. 选择保存位置和文件名

导出文件包含该分类及其所有按钮数据（`.kbar` 格式）。
:::

:::details 导出 UI 设置
在设置面板中：

1. 找到 **导出配置** 按钮
2. 选择 **UI 设置** 选项
3. 选择保存位置

导出文件包含界面主题、图标大小、布局等设置（`.json` 格式）。
:::

:::details 导出完整数据库
在设置面板中：

1. 找到 **导出配置** 按钮
2. 选择 **完整数据库** 选项
3. 选择保存位置

导出文件包含所有分类、按钮、图标、UI 设置等完整数据。
:::

## 数据导入

:::details 导入 KBar 数据库（.kbar/.kkbar/.zip）
Kkbar 支持导入 KBar 格式的数据库文件，方便从 KBar 迁移数据：

1. 在设置面板中点击 **导入** 按钮
2. 选择 `.kbar`、`.kkbar` 或 `.zip` 格式的文件
3. 预览导入内容，查看分类和按钮数量
4. 选择要导入的分类（支持全选/取消全选）
5. 点击 **导入** 按钮完成数据迁移

**支持的功能：**
- 自动解析 KBar 格式的 manifest.json 文件
- 自动提取并导入脚本（.jsx）、预设（.ffx）、图标等资源文件
- 支持智能套娃解压（自动识别嵌套的 .kbar 文件）
- 导入时采用智能合并模式，不会覆盖现有数据
- 支持分类、按钮、动作、图标等完整数据迁移
- 自动处理文件名冲突，生成唯一文件名
:::

:::details 导入 KBar 分类数据（.kbar）
1. 在设置面板中点击 **导入** 按钮
2. 选择要导入的 `.kbar` 文件
3. 预览导入内容，选择要导入的分类
4. 点击 **导入** 按钮，数据会智能合并到现有分类中
:::

:::details 导入 UI 设置（.json）
1. 在设置面板中点击 **导入** 按钮
2. 选择要导入的 UI 设置文件（`.json` 格式）
3. 确认后 UI 设置会被覆盖
:::

:::details 导入完整数据库（.json）
1. 在设置面板中点击 **导入** 按钮
2. 选择要导入的完整数据库文件（`.json` 格式）
3. 确认后所有数据会被覆盖
:::

:::warning 注意
- 导入完整数据库会**覆盖**所有现有数据
- 导入 UI 设置会**覆盖**当前界面配置
- 导入分类数据会与现有数据**合并**，不会覆盖
- 导入 KBar 数据库时，资源文件会自动提取到数据库目录
:::

## 数据备份建议

- 定期导出完整数据库作为备份
- 重要配置单独导出 UI 设置
- 可导出单个分类与他人分享
- 使用云备份功能自动同步（[高级功能](./03-cloud-backup.md)）

## 适用场景

- 备份数据库防止数据丢失
- 迁移到新电脑时转移配置
- 与他人分享单个分类的按钮配置
- 仅迁移 UI 设置保留按钮配置
