# 常见问题

## 安装问题

### Q: 安装后看不到 Kkbar 面板

**解决方法**：
1. 确认已用 [ZXP Installer](https://aescripts.com/zxp-installer/) 正确安装 `.zxp` 文件
2. 重启 AE，在菜单栏 **窗口 → 扩展** 查看是否有 Kkbar
3. 如果仍然看不到，请开启 [调试模式](/troubleshooting/02-debug-mode) 排查

---

### Q: 面板显示空白或加载失败

**解决方法**：
1. 关闭面板，重新打开
2. 确认 AE 版本为 2020 或以上
3. 开启调试模式，查看控制台是否有报错
4. 如果仍有问题，尝试卸载后重新安装

---

### Q: macOS 提示"无法验证开发者"

**解决方法**：
1. 打开 **系统设置 → 隐私与安全性**
2. 找到被阻止的应用，点击 **仍要打开**
3. 或终端执行：
   ```bash
   sudo xattr -r -d com.apple.quarantine /Library/Application\ Support/Adobe/CEP/extensions/com.kkbar.cep
   ```

---

## 使用问题

### Q: 如何创建第一个按钮？

**解决方法**：
1. 点击右上角 **设置**（齿轮图标）
2. 点击 **新增分类**，输入分类名称
3. 在分类下点击 **+** 添加按钮
4. 设置按钮名称、图标和动作类型
5. 回到主页，点击按钮即可执行

详细教程请参考 [快速上手](/guides/03-quickstart)

---

### Q: 如何为一个按钮设置多个动作？

**解决方法**：
1. 长按空格进入编辑模式
2. 右键点击按钮，打开动作设置面板
3. 为不同快捷键组合设置不同动作：
   - **单击**：基础动作
   - **Ctrl + 点击**：备用动作
   - **Alt + 点击**：第三种动作
4. 按下对应快捷键时，按钮图标会自动切换

详细教程请参考 [动作设置](/features/02-actions)

---

### Q: 动作执行失败怎么办？

**解决方法**：
1. 查看 Kkbar 日志面板中的错误信息
2. 检查脚本文件路径是否正确
3. 确认 AE 中是否有选中图层（部分动作需要）
4. 查阅 [错误代码文档](/troubleshooting/03-error-codes)

---

### Q: 如何快速测试一段脚本？

**解决方法**：
1. 创建新按钮，动作类型选择 **Scriptlet**
2. 在代码输入框中输入脚本代码
3. 点击按钮即可立即执行
4. 无需创建 .jsx 文件，适合快速测试

---

### Q: 按钮太多不好找怎么办？

**解决方法**：
1. 使用分类功能，按用途分组管理
2. 为常用按钮设置快捷键，一键触发
3. 拖拽调整按钮大小和位置，突出重点
4. 使用收藏功能，快速访问高频按钮

---

## 数据问题

### Q: 如何导出/导入配置？

**导出**：
1. 打开设置面板，点击 **导出配置**
2. 选择导出类型（分类 / UI设置 / 完整数据库）
3. 选择保存位置

**导入**：
1. 打开设置面板，点击 **导入配置**
2. 选择 `.json` 或 `.kbar` 文件

详细教程请参考 [数据备份](/settings/01-data-persistence)

---

### Q: 如何云同步配置？

**解决方法**（付费版功能）：
1. 打开设置面板，找到 **云备份**
2. 登录 GitHub 账号
3. 点击 **备份到 Gist** 上传配置
4. 其他电脑点击 **从 Gist 恢复** 即可同步

详细教程请参考 [云备份](/settings/02-cloud-backup)

---

### Q: 数据库可以放在其他位置吗？

**解决方法**：
1. 打开设置面板，找到 **数据库路径**
2. 点击 **设置** 选择新目录
3. 可选择创建新数据库或复制现有数据

适合多设备用户将数据库放在同步盘中。
